# Контекст проекта: Telegram-канал «дневник по датам» (Шевченко)

> Это вводные для агента. Прочитай целиком перед тем, как что-то писать.
> Сначала проектируем по слоям и проверяем источник на ОДНОЙ записи — только потом гоним по всему сайту.

## 1. Что строим

Telegram-канал, который раз в день автоматически постит одну датированную запись из дневника/писем
писателя. К записи привязана реальная историческая дата и возраст автора в тот момент.

Формат поста (образец, ему следовать):

```
Тарас Шевченко. Лайфстайл      ← жирный заголовок
<текст записи / цитата>

1857 год, 12 июня              ← жирная дата оригинала
43 года                        ← жирный возраст автора в тот момент
```

(реакции, 🔥/❤️ и просмотры на образце — это нативная механика Telegram, мы её не трогаем)

## 2. Источник

- **Сайт:** «Ізборник» — litopys.org.ua, раздел Шевченко.
- **Что берём:**
  - **Том 5** — «Журнал» (Дневник), июнь 1857 – июль 1858. Записи **датированы** → основа канала.
  - **Том 6** — Письма (тоже датированы) → расширяет покрытие календаря за пределы дневника.
- **Технически:** статичные свёрстанные HTML-страницы (старый дизайн начала 2000-х), **без JS-подгрузки**.
  Значит хватит обычного «скачать страницу → распарсить», тяжёлый headless-браузер не нужен.
- **Язык:** русский XIX века (с вкраплениями украинского/польского). **Перевод НЕ требуется** — постим как есть,
  сохраняя оригинальную орфографию.
- **Важно про содержимое:** на сайте текст идёт с научным аппаратом — сноски, комментарии, разночтения.
  Нам нужен **только авторский текст** Шевченко. Редакторские сноски/комментарии в пост НЕ тащим
  (и по чистоте цитаты, и по правам — см. п.9).

## 3. Архитектура (5 коробок, слева направо)

Ключевое решение: **парсим сайт ОДИН раз к себе, дальше живём из своей базы.** Сайт в ежедневном
постинге не участвует вообще.

```
[ИМПОРТЁР]  →  [СЫРАЯ БАЗА + ручная вычитка]  →  [ДВИЖОК ВЫБОРА]  →  [РЕНДЕР]  →  [ПОСТЕР]
 (руками)          (даты, «одобрено»,            (стратегии +        (шаблон     (+ журнал
                    «интересная»)                  фолбэк)            поста)      опубликованного)
                                                         ↑__________________________|
                                                          антиповтор читает журнал

                         [ПЛАНИРОВЩИК] раз в день дёргает: движок → рендер → постер
```

1. **Импортёр** — парсит litopys → кладёт записи в нашу базу. Запускается руками. Должен быть
   **идемпотентным**: повторный запуск до-заливает новое и НЕ плодит дубли (см. п.4, поле `source_id`).
2. **Сырая база + вычитка** — между импортом и движком стоит ручной шаг человека: проверить текст,
   проставить дату, отметить `approved`, отметить `interesting` у годных цитат. Движок берёт только `approved`.
3. **Движок выбора** — сердце. Цепочка стратегий с фолбэком (см. п.5).
4. **Рендер** — запись → готовый текст поста по шаблону из п.1.
5. **Постер** — отдаёт текст в канал (Telegram Bot API, бот — админ канала, `sendMessage`,
   `parse_mode=HTML`) и пишет факт публикации в **журнал**, который читает антиповтор.
6. **Планировщик** — снаружи, раз в день дёргает цепочку (cron / GitHub Actions / облачная функция).

Почему так: ежедневный постинг не зависит от того, жив ли сайт / не поменялась ли вёрстка; хрупкий
парсер запускается под присмотром, а не в 9 утра без свидетелей.

## 4. Модель записи (нормализованная)

Единый формат, к которому импортёр приводит всё. Движок видит только это, не зная про устройство сайта.

| поле           | смысл                                                        |
|----------------|-------------------------------------------------------------|
| `id`           | наш внутренний id                                           |
| `source_id`    | **стабильный признак записи на сайте** (URL + якорь / хеш текста) — для идемпотентности |
| `author`       | «Тарас Шевченко»                                            |
| `text`         | авторский текст (без редакторских сносок)                   |
| `orig_date`    | дата оригинала: год / месяц / день (может быть НЕ полной у писем) |
| `kind`         | `diary` / `letter`                                          |
| `approved`     | прошло вычитку (да/нет)                                     |
| `interesting`  | флаг «годная цитата» — для стратегии рандома                |
| `published_at` | когда запостили (пусто = ещё нет) — основа антиповтора      |

**Возраст НЕ храним** — считаем на лету: `возраст = f(orig_date − дата_рождения)`.
Дата рождения Шевченко — константа: **25 февраля 1814 (ст. ст.) / 9 марта 1814 (н. ст.)**. Агенту: сверить.

## 5. Движок выбора — цепочка стратегий

Просто упорядоченный список правил. Пробуем по очереди, **первое сработавшее побеждает**.
Добавить новую стратегию = дописать одно правило, не трогая остальное.

1. **«В этот день»** — найти `approved`-запись, где день/месяц `orig_date` == сегодня. Если несколько —
   правило выбора (напр. ещё не публиковалась / самая ранняя по году).
2. **Фолбэк: рандомная интересная цитата** — если на сегодня по дате ничего нет, берём случайную
   из `approved` + `interesting`, ещё не опубликованную.

Сквозные требования к движку:
- **Антиповтор** — не постить то, что в журнале уже опубликовано.
- Что делать, если вообще нечего (всё опубликовано / буфер пуст) — пропустить день + алерт (не постить кашу).

## 6. Подводные камни (прочитать обязательно)

- **Календарь ст./н. стиль.** Шевченко датировал по старому стилю (юлианскому). НЕ конвертируем в
  григорианский — **постим дату буквально как написана в источнике** («12 июня» → постим 12 июня).
  Это и проще, и совпадает с ожиданием читателя. Конвертацию не закладываем.
- **Неполные даты у писем.** Часть писем датирована только месяцем/годом или приблизительно. Такие
  не годятся для стратегии «в этот день» — пусть идут только в пул рандомных цитат (или помечаются).
- **Хрупкость парсера.** Адаптер цепляется за вёрстку. Изолировать его от всего; при импорте —
  проверка «похоже ли вытащенное на нормальную запись», иначе пропустить + лог. Ломается только импортёр, не движок.
- **Вежливость к сайту.** «Ізборник» — некоммерческий академический проект. Парсить мягко: пауза между
  запросами, кешировать скачанное локально, в идеале один проход. Не долбить.
- **Идемпотентность.** Без `source_id` повторный импорт растащит базу на дубли. Тащить стабильный
  признак сразу при первом парсинге.
- **Telegram-разметка.** `parse_mode=HTML` надёжнее MarkdownV2 (меньше боли с экранированием).
  В авторском тексте могут встречаться символы, ломающие разметку — экранировать тело поста.

## 8. С чего начать (первый шаг агента)

**НЕ** писать сразу весь импортёр. Сначала:

1. Скачать **одну** страницу дневника с litopys.org.ua и показать пользователю, что реально извлекается:
   - вытащился ли **текст записи** чисто (без редакторских сносок),
   - вытащилась ли **дата** в машинном виде (а не только внутри текста),
   - есть ли **стабильный `source_id`** (URL+якорь).
2. На основе одной записи честно сказать: заработает ли «в этот день», или канал поедет в основном на рандоме.
3. Только убедившись на одной — проектировать проход по всему тому.

## 9. Права (коротко)

Собственные тексты Шевченко (дневник, письма) — автор умер в 1861 — **общественное достояние**, постить
свободно. А вот научный аппарат издания (комментарии, сноски, разночтения) — работа современных
редакторов/Академии, со своими правами. Ещё одна причина брать **только авторский текст** и оставлять
редакторские примечания за бортом.

## 10. Scope — чего пока НЕ делаем

- Не делаем второго автора / ротацию — сначала один канал, один Шевченко.
- Не делаем сложную конвертацию календарей.
- Не делаем веб-морду/админку — вычитка руками прямо в таблице.
- Не автоматизируем сбор «на лету» — только разовый импорт + жизнь из базы.
