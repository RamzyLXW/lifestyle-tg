# Lifestyle TG — «Тарас Шевченко. Лайфстайл»

Телеграм-канал, который раз в день постит датированную запись Шевченко (дневник/письма/стихи)
+ дату оригинала + возраст автора. Спека — в [BRIEF_shevchenko_bot.md](BRIEF_shevchenko_bot.md).

Принцип (бриф, п.3): **парсим litopys.org.ua ОДИН раз к себе → дальше живём из своей базы.**

## Стек / решения

- **Node 22 + TypeScript** (переиспользуем `googleapis`/Telegram-обвязку из `~/LuxWinWeb/telegram-to-sheets`).
- **База: локальный SQLite** (`shevchenko.db`, встроенный `node:sqlite`) → позже **экспорт в Google Sheets** для ручной вычитки (бриф п.10).
- **Google-доступ:** service-account `telegram-bot@steady-habitat-485620-a6.iam.gserviceaccount.com`
  (ключ в `~/LuxWinWeb/telegram-to-sheets/…json`). Чтобы писать в таблицу — расшарить её на этот email (Editor).
- Источник в **windows-1251**, авторский текст vs аппарат отделены CSS-классом: `p.K1` (текст) / `p.Prym` (сноски — НЕ берём).

## Запуск

```bash
npm install
npm run import -- <url>                  # парс одной страницы → SQLite (без аргумента: тестовая shev104)
npm run import:volume -- 5 --max 400     # обход тома по навигации (отсев аппарата)
npm run import:diary                     # разрезать дневник shev501 на датированные записи
npm run import:volume -- 1 --max 250     # поэзия тома 1 (Кобзар), 2 — для тома 2
npm run excerpts -- shev503              # эвристич. цитаты-кандидаты (фолбэк, без ИИ)
npm run curate -- --limit 5             # АГЕНТ-КУРАТОР: Claude предлагает выжимки по RUBRIC.md (нужен ANTHROPIC_API_KEY)
npm run curate -- --list                # что будет курировано — без обращения к API
npm run render                           # превью поста (оба варианта низа)
SHEVCHENKO_SHEET_ID=<id> npm run export      # SQLite → Google Sheets (для вычитки)
SHEVCHENKO_SHEET_ID=<id> npm run import:sheet # Google Sheets → SQLite (вернуть вычитку)
npm run engine -- --date 18.02           # выбрать запись на день + собрать пост (без отправки)
npm run post -- --check                  # проверить токен бота и доступ к каналу
npm run post -- --dry-run                # выбрать пост на сегодня, показать, НЕ отправлять
npm run post                             # выбрать на сегодня и ОТПРАВИТЬ в канал
npm run typecheck
# секреты — в .env (TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, SHEVCHENKO_SHEET_ID, ANTHROPIC_API_KEY), файл в .gitignore
```

## Сделано (разведка, бриф п.8)

На ОДНОЙ странице (`shev104`, стих «ДУМКА») проверено и работает:
- авторский текст извлекается чисто (≈20 абзацев редакторского аппарата отброшены по классу `Prym`);
- дата в машинном виде (здесь только год `1838` → контент для рандом-пула, не для «в этот день»);
- стабильный `source_id` (имя страницы) → импорт идемпотентен (повторный запуск дубля не создаёт).

## Проверено сквозняком (на 2 записях)

`import → export(Sheets) → ручная вычитка → import:sheet → engine → render`. Движок:
- «в этот день» (`--date 18.02` → shev503, footer date «1860 год, 18 февраля / 46 лет»);
- фолбэк случайная интересная (сегодня → shev104, footer source «Думка»);
- антиповтор (после `published_at` запись выпадает из пула).

## Структура Тома 5 (важно)

Том 5 = не «страница=запись», а смесь: `shev501` Щоденник (ОДНА страница → режется сплиттером на 298 записей),
`shev502` Автобіографія, `shev503` Письмо, `shev504–512` статьи/работы, `shev513` Варіанти,
`shev514–524` Коментарі (аппарат, НЕ берём), `shev525–529` указатели. Цепочка «Наступна» замкнута (shev5→…→shev529→shev5).

## В базе сейчас

~530 записей: **222 стиха** (Кобзар, тома 1–2, kind=poem — ОСНОВНОЙ источник, нативно укр.),
**298 дневниковых** (датированы, kind=diary, рус.), 9 статей/работ, 1 письмо.
Дневник почти сплошь покрывает 1857-06…1858-07 (естественные пропуски, напр. 22–24 июня 1857).
Стихи в основном с годом (`date_precision=year`) → пул рандома, не «в этот день»; у безымянных стихов
`work_title` = первая строка (берётся из `<title>`). Часть страниц — циклы (напр. shev201 «В казематі»):
несколько стихов на одной странице — куратор режет на один фрагмент. 3 стиха рус. (shev121 «Слепая»,
shev123 «Тризна», shev151) → пойдут через перевод.
**Курирование сделано (вся укр. поэзия): 157 фрагментов в пуле `interesting=1`, 62 «мимо», у всех `excerpt`+
`source_label`+`footer_kind=source`, дословность проверена.** Человеческого `approved=1` пока нет (следующий шаг).

## РЕШЕНО: стратегия контента (см. RUBRIC.md)

- **Язык поста — украинский.** Основной источник — **стихи / «Кобзар» (тома 1–2)** (нативно укр., quote-режим).
- **Русский Дневник/письма** — выборочно и через **литературный перевод на украинский** (новый шаг). Дневник вторичен.
- **Темы** — вечные житейские; грустное/тяжёлое ок (гнёт — часть голоса); не сводить к застольям (Шевченко ≠ Чехов).
- Человек ставит только `approved=1`; остальное (excerpt, перевод, источник, низ) — система/агенты.
